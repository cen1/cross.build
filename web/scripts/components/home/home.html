Cross platform code building and testing

Cross.Build allows you to build code on several platforms (operating systems) at once. Just provide your git repo and build commands for each platform.
Main target are projects written in compiled languages (C, C++). This project is in a proof of concept stage. You can use this demo page to test but it can get wiped anytime.

Supported platforms

  • Ubuntu 14.04
  • Freebsd 10

How does it work?

  1. Add your project by specifying your git repo url and build commands for each selected platform.
  2. Cross.Build finds free build servers for each platform and creates a new container on each (Docker, Jail).
  3. Containers are added as nodes to Jenkins.
  4. A new Jenkins job is created for each platform. Each job executes in it's own container
  5. Jenkins builds your project on each platform and you can view the results on Cross.Build frontend. Changes are detected by SCM polling.
  6. Servers are constantly monitored for their load and new ones are automatically provisioned as the average load approaches the limit. EC2 is used as the provider.
    This means that build servers will automatically scale with the demand as new projects are added. No need to spin up the servers by yourself.

Design decisions

  • Build commands are stored in the database, not in project files. No need for CI specific bloat in your repo.
  • You have sudo access inside your containers. No preinstalled dependencies. You decide.
  • Only platforms available at major cloud providers and those with container support are avalible at the moment. This means Linux and FreeBSD.
    Windows Server 2016 is the next target once it is released with Docker support.





Technologies used



Backend in it's entierty is developed in Java-EE 7 and Java 8. Load monitoring component is developed in Kumuluz-EE framework due to it's smaller footprint.

Jenkins is used as the backend CI provider and is managed by Cross.Build via REST API. At the moment only a subset of Jenkins functionalities are exposed via Cross.Build REST API and frontend. More will be added as the project develops.

WildFly 10 is used as the Java-EE application server for project deployment.

Frontend is developed in AngularJs framework and uses Cross.Build REST API for all communication.

JClouds Java library is used for provisioning of new virtual machines and command execution in combination with custom bash scripts for server and container setup.

Keycloak is used for authentication and authorization of users. Keycloak JavaScript library is used in combination with AngularJs for seemles integration of user registration and login. GitHub is used as OAuth provider of the demo page.

Docker containers are used as Jenkins nodes on Linux platforms. FreeBSD Jail containers are used as Jenkins nodes on FreeBSD platform. Ezjail tool with base template and pf are used to manage the creation of new Jails.

All build servers are currently provisioned on AWS EC2 via JClouds library. Due to platform agnostic nature of JClouds it will be possible to support additional cloud providers in the future. EC2 was chosen as the initial provider due to pricing and platform availability through AWS marketplace.
